HTML

  • Some of the most commonly used elements to add content on a webpage are:
    • Text elements
    • Image elements
    • Header elements
    • List Elements
  • Some of the most commonly used elements to divide and describe a page are:
    • Meta-related elements
    • Content division elements
    • Semantic elements
  • HTML elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigate the webpage.
  • HTML builds the sturdy "frame" of the webpage that everything else is built on top of.
  • The head element contains information, or metadata, that helps the browser to render the page correctly.
  • Elements included within the head element are not visible when a user looks at a webpage.
  • The body element represents the visible content shown to the user.
  • Some common metadata tags are:
    • title
    • style
    • base
    • link
    • meta
    • script
  • The charset element sets our character encoding.
  • The http-equiv element sets an HTTP header for content.
  • The viewport element sets the defalut view for a webpage.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are three ways to style a webpage with CSS: inline, internal, and external.
  • Inline CSS is used when a developer wants to style an element in the HTML file. They're useful for quick and permanent changes but are less flexible than external or internal style sheets, because each inline style must be edited individually whenever you want to make a change.
  • Internal CSS is used to embed an entire style sheet in the HTML file using a style elemenent inside the head element.
  • External CSS is the most commonly used because it allows developers to keep all their CSS rules in a separate file, making design easier.
  • When using an external style sheet, you link the file using a link element placed in the HTML's head element.
  • A CSS rule is broken up into a selector, and a declaration block inside of two curly brackets, for example: div {color: blue; text-align: justify;}
  • A selector defines the element or attributes to which the rules or declarations apply.
  • A declaration contains two components: the CSS property we want to apply and the value of the property.
  • The declaration property is followed by a colon and then the value of the property is followed by a semicolon. For example: color: blue;
  • Two or more elements that will share the same styling can be grouped together, which cleans up our code with less repetition.
  • Assigning colors in CSS can be done in many ways, including semantic references like gray or blue but you can also refer to a colors hecadecimal code or hex #.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • Here's a link to a cheat sheet of git commands: https://education.github.com/git-cheat-sheet-education.pdf
  • Use Markdown language to add issues to a GitHub repository
  • CLI Commands:
  • ls - list of all files and folders in the current working directory
  • cd - change directories, for example: "cd Downloads"
  • pwd - print working directory, will show the name of the current directory
  • cd .. - allws us to navigate backwards one directory at a time
  • mkdir - make directory, for example: mkdir Prework
  • code. - opens our code editor program
  • An SHA, or hash, is assigned every time we save a version of our work using version control. This allows us to keep a record of specific changes committed. The unique identifier gives us a reference to that specific version of code so it can be retrieved as needed.
  • A branch is a copy or version of the code that can be worked on independently in isolation so that changes don't disrupt other team members who are also working on the code.
  • A new branch adding something to the project is called a feature branch.
  • In the feature branch, changes are made to the code that will be committed. Commit messages are then added to describe what changed in the code.
  • Then the changed branch is pushed to our GitHub repository.
  • Then a pull request, or PR, is used to merge the changed branch into the main branch.
  • When working with others, the PR would be checked and then approved or denied.
  • git add -A - this command adds all modifications in the current working branch to the staging area
  • git commit -m "message text here" - this command adds a message that describes what the commit contains
  • git pull origin maing - this command pulls down any changes from the main branch to our current working branch
  • git push origin feature/your-text-here - this command pushes the changes we have locally to our remote GitHub branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • The variable keyworkd "const" allows us to create variables that can't be reassigned a value. Once a variable is defined using const it is essentially "locked down".
  • console.log method outputs a message to the web console by adding an argument. For example: console.log(topics);
  • An array is a single variable used to hold a group of data that is typically related in some way.
  • Arrays are not limited to contain only one type of data. A single array can store strings, numbers, booleans, and so on.
  • You access data inside an array by knowing its location or index. Indices in JavaScript are zero-indexed and sequential. Meaning they always start at 0 and then 1 then 2 and so on.
